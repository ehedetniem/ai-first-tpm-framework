# Program Lifecycle

Use this lifecycle for cross-functional technical programs.

## Phase 0 - Intake

Objective:
- Determine whether incoming work is a program, project, or request.

Exit criteria:
- Problem statement is clear.
- Owner is assigned.
- Initial scope boundary exists.

## Phase 1 - Shape

Objective:
- Build execution clarity before broad launch.

Exit criteria:
- Timeline draft exists.
- Roles and stakeholders are mapped.
- Top risks and dependencies are identified.
- Decision backlog is started.

## Phase 2 - Build and Coordinate

Objective:
- Drive execution and maintain cross-team alignment.

Exit criteria:
- Weekly status cadence is stable.
- Decisions are captured via ADRs.
- Risk mitigations are active.

## Phase 3 - Land and Scale

Objective:
- Validate outcomes and scale adoption.

Exit criteria:
- Success criteria are measured.
- Operational handoff is prepared.
- Remaining critical risks are addressed.

## Phase 4 - Run State

Objective:
- Sustain outcomes through steady-state governance.

Exit criteria:
- Ownership model is stable.
- Monitoring and review cadence is operational.
