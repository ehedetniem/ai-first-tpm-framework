# Contributing

Thanks for helping improve the AI-First TPM Framework.

## Contribution scope

Useful contributions include:
- New or improved templates for TPM artifacts
- Lifecycle and governance refinements
- Adoption guidance and implementation examples
- Clarifications that improve reusability across industries

## How to contribute

1. Fork the repository.
2. Create a feature branch.
3. Make focused changes with clear rationale.
4. Update docs when behavior or guidance changes.
5. Submit a pull request with:
   - Problem statement
   - Proposed change
   - Why it improves framework usability

## Quality expectations

- Keep language practical and implementation-oriented.
- Avoid references to internal/private systems.
- Prefer reusable guidance over organization-specific patterns.
- Preserve the human-in-the-loop governance stance.
- Keep guidance Copilot-native and prompt-first; position direct CLI steps as fallback when both paths exist.

## Style guidelines

- Use short sections and scannable lists.
- Keep templates concise and actionable.
- Use consistent status language where applicable.

## Pull request checklist

- [ ] Change is clear and scoped
- [ ] Docs are updated
- [ ] Template changes include usage notes
- [ ] No confidential data or internal identifiers
- [ ] Governance boundaries remain explicit
